HTML

  • the head element contains information about the webpage
  • the body element contains information visible to the user
  • the title element is displayed in the browser tab
  • the h1 element is the largest heading
  • the p element is used for paragraphs
  • meta elements are child elements of the head element and give the browser "metadata" information
  • attributes like 'lang="en"' give the browser information about the element and are located in the opening tag of an element

CSS

  • CSS defines the "styling" or look of HTML structured content
  • CSS can be inline or contained in a internal or external stylesheet
  • External stylesheets are preferred
  • External stylesheets are linked in the <head> element using a <link> element with attributes of:
    • rel="stylesheet" - specifies the relationship between the current document and the linked document
    • href="path-to-stylesheet.css" - specifies the location of the linked document
  • CSS "selectors" define the element or attribute to which the CSS declarations (rules) will apply
  • CSS declarations are contained within curly braces and consist of a property and value separated by a colon
  • "margin" defines space OUTSIDE an element
  • "padding" defines space around the content INSIDE an element and makes the element BIGGER
  • Every HTML element is a box, consisting of margins, borders, padding and content - sometimes referred to as the CSS Box Model
  • The "*" selector is a wild card meaning the declarations apply to all elements on a page
  • Combining CSS rules using multiple selectors helps to write DRY code (Do Not Repeat Yourself) to reduce the size and complexity of a codebase
  • display: block; starts the selected element on a new line
  • class attributes can assign CSS rules to multiple elements

Git

  • git status - gives info on the current branch
  • git checkout <branch-name> - switches to the specified branch
  • git checkout -b <new-branch-name> - creates a new branch and switches to it
  • git switch <branch-name> - switches to the specified branch - intended to be a more intuitive command than git checkout
  • git add -A - stages all files in the working branch to be committed to github
  • git commit -m "commit message" - commits the staged files to github
  • git pull origin main - sync local branch with remote branch in Github - note "origin" is the default name for the remote repository from which a local repository was cloned

JavaScript

  • JavaScript (JS) is a dynamic programming language that allows users to interact with websites
  • JS takes static html/css and makes it interactive
